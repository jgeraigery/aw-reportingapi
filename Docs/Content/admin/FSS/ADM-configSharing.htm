Configuring link sharing for the system

Links are a quick, easy way for users to share their HCP Anywhere files and folders. If link sharing is enabled, users can create links in the HCP Anywhere User Portal, desktop application, mobile application, and Outlook Add-in.

You can enable or disable link sharing for the entire system. Once link sharing is enabled for the system, you can also configure link sharing for the profiles of your choice, which offers you additional control over who can share and how. For information on:

•Creating profiles, see Creating a profile.

•Editing a profile, see Editing file sync and share settings in a profile

Internal vs. public links

•Internal links — Internal links are only accessible by people that have user accounts from the Active Directory forest or SAML identity providers that HCP Anywhere knows about. Effectively, this includes everyone in your organization. Users need to provide valid login credentials to access internal links.

•Public links — Public links are accessible by anyone. A user does not need to provide any login credentials to access a public link.

Read vs. upload links

When someone creates a link to a folder, they can choose whether the link is read only, upload only, or both (if enabled). When someone creates a link to a file, the link is always read only.

•Read permissions — When a link has the read permission, someone who accesses the link can view and download the contents of the file or folder that the link points to.

•Upload permission — When a link has the upload permission, someone who accesses the link can upload files to the folder that the link points to.

When configuring link sharing for the system, you enable and disable read and upload permissions together. When configuring link sharing for a profile, you can be more specific about the types of links that the users in profile can create.

File preview

When someone accesses a link, they can see the name of the file or files that the link points to if the link has the read permission. You can also enable file preview for your users. When file preview is enabled, a person who accesses a link can see a preview of the file contents. File preview is available for files under 5 MB with these extensions:

•png

•jpeg

•wav

•mp3

•mp4

For audio and video files, the file can only be previewed if it is under 5 MB. For PDF files, the file can only be previewed if the browser allows PDF files to be previewed. Some browsers do not allow PDF files to be previewed.

To configure link sharing for the system:

1.In the top-level menu, mouse over the File Sync and Share tab to display a secondary menu.

2.In the secondary menu, click on Configuration.

3.On the Configuration page, click on Sharing.

4.On the Sharing tab:

oTo enable or disable internal link sharing, select or deselect the Enable all link sharing (read and upload) option.

When you disable link sharing, all existing links are deactivated. Reenabling link sharing reactivates all existing links.

oTo enable or disable public link sharing, select or deselect the Enable all public link sharing (read and upload) option.

When you disable public link sharing, all existing public links are deactivated. Reenabling public link sharing reactivates the links.

oIn the Default way to share field, select Internally or Publicly.

oTo enable or disable file preview, select or deselect Enable file preview for shared link reads.

5.Click on Update Settings.
